Filters

Filters methods are meant to filter through a list of locators resuling from a find method or native playwright method.

Filters are meant to be ‘low level’. If you need an advanced and quick way to search for an item, use the ‘find_*’ methods, which are designed to raise Exceptions and ensure a return.

find_all_by_attr(selector: str, attr: str, text_content: str) → list[playwright.sync_api.Locator][source]

Finds all locators associated to the passed selector that has the ‘text_content’ in its ‘attr’ attribute. To search for any selector type, you can set ‘selector’ to “*”.

find_one_by_attr(selector: str, attr: str, text_content: str) → playwright.sync_api.Locator[source]

Finds the first element associated to the passed selector that has the ‘text_content’ in its ‘attr’ attribute. To search for any selector type, you can set ‘selector’ to “*”.

get_by_content_text(text: str, selector: str = 'div', strict: bool = False, n_min: int = 0, timeout: int = 2000) → list[playwright.sync_api.Locator][source]

Get a list of Locator(s) which contains the ‘text’ arguments. Be careful, technically, if you have a span with your text argument, inside a div and inside another div, they will all get appended to the list.

Parameters:

  • text (-) – the string that should match.

  • selector (-) – tag in which the search is made. Can be set to “*” for global search.

  • strict (-) – if True returns element strictly matching string.

  • n_min (-) – minimum elements to wait for (raise exception if less)

accept_cookies(text: str = '', case_sensitive: bool = False, recursive: bool = True, click_count: int = 1, selector: str = 'button') → None[source]

Helper to bypass the “accept cookies” pop ups. If no ‘text’ is specified, then it clicks every buttons that would be a good candidate. If text is specified but the buttons is not found an error will be raised. If the “text” gets multiple candidates, then they will all be clicked unless the “recursive” parameter is changed to False.

Parameters:

  • text (-) – the text that identifies the accept button.

  • case_sensitive (-) – wether or not the search is case sensitive.

  • click_count (-) – as those popups appears at dom loading, you might find cases where clicking multiple times gets the focus correctly.

  • recursive (-) – wether or not we want to click on every candidate buttons.

  • selector (-) – Chose in which selectors the button can be found with.

Go to the desired url and assure the page is fully loaded in a timely manner.

Parameters:

scroller(scroller_depth: int = 0, axis: Literal['y', 'x'] = 'y', reload_timeout: int = 2000, max_reload: int = 0, cond: dict = {}, speed: int = 50) → None[source]

scroll all the way down a scroll bar.

Parameters:

  • scroller_depth (-) – which depth should the scrollbar be at, 0 being the full doc, 1 the first scrollable container inside the full doc, 2 the second … etc.

  • axis (-) – scrolling axis.

  • reload_timeout (-) – for dynamically generated content. If a value is set, waits ‘reload_timeout’ ms and check if scroll_bar is still maxed.

  • max_reload (-) – maximum reloading allowed. -1 is for reload until finished.

  • attribute (-) – which attribute to match the scroll_bar.
